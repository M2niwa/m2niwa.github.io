Hermes 的自我进化：Skills 和 Memory 如何让 AI 越用越强

2026-06-16
Hermes Agent 的核心差异在于它的「封闭学习循环」——Skills 让工作流可复用，Memory 让偏好跨会话持久化。深入拆解这两个系统的工作原理和实战用法。

上次讲了 Hermes Agent 的安装和基础使用。用了几个星期后，真正让我觉得它和其他 AI 工具有本质区别的，是它的学习和进化能力——Hermes 不是每次对话都从零开始的失忆 chatbot，而是一个会积累经验、改进工作流、越用越顺手的 agent。这背后依赖两个核心系统：Skills 和 Memory。

Skills：Agent 的「肌肉记忆」

在 Hermes 的设计里，Skill 是一个可复用的知识包——包含完成某类任务所需的所有信息：步骤、命令、常见陷阱、验证方法。每次 Hermes 成功完成一个复杂任务后，它可以自动创建或更新一个 Skill。下次遇到类似任务时，Skill 会被加载到上下文中，Agent 不需要重新「思考」该怎么做。

Skills 使用高效的 token 加载模式：只有描述（description）在会话启动时注入 prompt，完整内容在 Agent 判断需要时才加载。这避免了每次都把大段技能文档塞进上下文窗口。

Skill 的另一个设计巧思是它的 跨平台分发能力——通过 agentskills.io 或 GitHub 仓库，你可以安装社区贡献的 Skill。社区已经有覆盖各种场景的 skill pack：天气数据抓取、邮件总结、Nextcloud 部署、甚至用 Hermes 给孩子讲睡前故事。

我最常用的 Skill 是自己迭代出来的 personal-website——里面记录了网站的项目路径、CSS 变量约定、常见 JS 陷阱（比如 let 重复声明导致 SyntaxError）、部署流程。每次修改网站时，Agent 自动加载这些上下文，不再犯已经踩过的坑。

Memory：跨越会话的持久记忆

如果说 Skills 是「怎么做」，Memory 就是「关于你和你的环境的一切」。Hermes 的 Memory 系统维护两个文件：

  • MEMORY.md：项目路径、工具偏好、环境配置、经验教训
  • USER.md：用户名、联系方式、GitHub 账户、语言偏好

每次新会话启动时，这两个文件的内容会被注入系统 prompt 作为固定区块。Memory 有严格的字符限制（MEMORY.md 约 2200 字符，USER.md 约 1375 字符），防止系统 prompt 膨胀。

一个容易被忽视但很重要的设计：Hermes 不是被动记录一切，而是有选择地保存——它会在以下时机自动触发记忆写入：

  • 用户表达了明确的偏好或纠正了 Agent 的行为
  • 发现了新的环境细节（新项目路径、工具版本）
  • 完成了复杂任务并提炼出可复用的经验

还有一个安全细节：Memory 写入前会经过注入扫描——检测 prompt injection、凭证泄露、不可见 Unicode 字符等威胁模式。因为 Memory 内容会被注入系统 prompt，这个防护是必要的。

封闭学习循环的实际效果

Skills + Memory 的组合形成了一个 封闭学习循环：每次交互 → Agent 学习新知识 → 保存到 Memory 或创建/更新 Skill → 下次交互时自动加载 → 表现更好。

实际体验上，用 Hermes 两周后能明显感受到变化：它不再需要我重复「我的项目在 D 盘」「用 Python 3.13」「网站端口是 8766」——这些信息已经存在 Memory 里。更重要的是，当它犯了一次错误后，会把修正方案写入对应的 Skill，保证同样的错误不会出现第二次。

这就是 Hermes 和 ChatGPT / Copilot 最本质的区别：它不是每次从零开始的对话工具，而是一个持续进化的开发伙伴。
