Hermes Agent 初体验：一个能记住你的终端 AI

2026-05-18
从零安装 Hermes Agent 到完成第一个实战任务的完整记录——由 Nous Research 打造的自主 AI agent，它不止聊天，还会记住你的偏好、创建技能、跨会话持续进化。

最近开始用 Hermes Agent，由 Nous Research（就是训练 Hermes 系列开源模型的那个团队）开发。它不是 LangChain 套壳，也不是 Copilot 那种嵌在 IDE 里的补全工具——它是一个完全自主的终端 AI agent，有记忆、会学习、能自己写 skill 并在后续对话中复用。

这篇文章记录从零安装到跑通第一个实用任务的完整过程。

安装：比想象中简单

官方提供了一行命令安装：

# Linux / macOS / WSL2
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

# Windows (PowerShell)
iex (irm https://hermes-agent.nousresearch.com/install.ps1)

装完后用 hermes setup --portal 完成 OAuth 配置。这一步直接搞定三件事：接入模型提供商（默认 Nous Portal）、开通 Tool Gateway（web 搜索、图片生成、TTS、浏览器自动化四项工具）、生成 API key。全程大约 60 秒。

Hermes 把配置分成两层：config.yaml 放普通参数（模型、工具开关），secrets.yaml 放敏感信息（API key、token）。这种分离比把所有东西堆在一起清爽得多——至少不会不小心把 token 提交到 GitHub 上了。

第一次对话：不只是聊天

启动后你会看到一个终端 TUI，顶部是模型名、可用工具和已加载的 skills。Hermes 的默认行为就和其他 AI 聊天不同——它默认拥有终端执行权限。你不是在「问它问题」，而是在「让它做事」。

我试的第一个任务：

$ hermes
> 帮我在当前目录初始化一个 Python 项目，用 FastAPI 写一个 hello world

Agent 的动作序列：

  1. 读取当前目录结构（ls）
  2. 创建 main.py（write_file）
  3. 创建 requirements.txt（write_file）
  4. 运行 pip install -r requirements.txt（terminal）
  5. 启动 uvicorn 验证服务响应（terminal）
  6. 完成后汇报结果

没有「建议你这样写代码」的废话——直接读到、写到、跑到、验证完。这个执行闭环在传统聊天式 AI 里是做不到的。

最惊喜的功能：对话可以「恢复」

Hermes 有个细节设计让我印象很深——会话持久化。你在终端里输 /new 开启新对话，用 /resume 回到之前的会话。跟 AI 的对话不再是「一次性」的——它可以跨多次终端打开延续。

这在处理复杂多步骤任务时特别有用。比如你下午写了一半的 API 接口，晚上回来 /resume，Agent 记得你做到哪了、文件路径、测试结果——不需要重新交代一遍上下文。

文档里提到 Hermes 支持 Memory 系统，分两个文件：MEMORY.md 存环境和经验，USER.md 存用户身份和偏好。每次新会话启动时，这些内容会被注入系统 prompt。我在第一周还没深入用这个功能，但已经在每次交互中感受到 Agent 在「学习」我的工作习惯。

安全和隔离

Hermes 的终端能力是它的核心优势，但也是最大的安全考量。官方提供了几种隔离方案：

  • Docker 容器模式：Agent 在独立容器内执行命令，不影响宿主机
  • SSH 远程模式：Agent 通过 SSH 连接到远程服务器工作
  • 命令审批模式：高危操作（rm、sudo）需要用户手动确认

我用的是 Docker 隔离——配置一行 backend: docker 就够了。Agent 的所有文件操作、pip 安装、服务器启动都在独立容器里，即使出错也不会影响宿主机环境。
